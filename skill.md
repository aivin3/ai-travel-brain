# /travel — 旅遊研究・規劃・路書 完整 Workflow

> 觸發：用戶講「幫我計劃去 X 旅行」/「研究下 X 點玩」/「整份 X 路書」/ 貼一張行程表叫整靚。
> 輸出語言繁中；景點/酒店/餐廳保留當地原名。完成必出 `roadbook.html`。
> 配合 `CLAUDE.md` 鐵律執行。

---

## Step 1 · 資訊收集
確認最小必要欄位，缺就問（一次過問齊，唔好逐條）：
- 出發地（預設香港）、日期或日數、人數（含小朋友/長者年齡）
- 目的地（已定？定要我推薦？）、預算範圍、偏好（親子/美食/自然/購物/文化/慢遊…）、特別需求（行動不便/飲食禁忌）

行程已有（用戶貼表/圖/文字）→ 跳去 Step 4 intake。

## Step 2 · 目的地推薦（只在用戶要推薦時）
- **A 候選**：按季節、人群、出發地，WebSearch 搵 3-5 個候選。
- **B 交通可行性**：篩走單程交通過長（理想 ≤4-5 小時直達；轉乘多要標明）。
- **B2 天氣**：查旅行月份天氣，剔除雨季/極端天氣高風險。
- **C 比較表**：出多維表（飛行時數 / 預算 / 親子友善 / 天氣 / 特色）俾用戶揀，唔幫佢決定。

## Step 3 · 多維 research（並行）
對選定目的地，並行查（可開多個 subagent）：
1. **天氣**：旅行日期 high/low、降雨機率、穿搭建議。
2. **交通**：去程/回程（航班或火車）+ 市內交通（地鐵/交通卡）+ **多段轉乘驗證**（轉乘時間、尾班車、站間步行）。
3. **景點**：票價、開放時間、預約需要、親子適合度；附來源。
4. **餐飲**：就每日所在區搵餐廳（菜系、人均、招牌、位置）。
5. **住宿**：2-3 間，比位置/價/評分；睇 Booking/Agoda/Google Hotels。
6. **真實體驗 / 貼士**：當地實用注意（退稅、插頭、SIM、緊急電話、小費文化）。

> 每個事實附 ≤120 字逐字引用 + 來源；價標可靠度（實查/參考~/估算）。

## Step 4 · 行程編排
- **地理分群**：同區景點放同一日，減少折返。
- **節奏交替**：動靜結合，避免一日全行山/全博物館。
- **餐廳就近**：午晚餐揀當日活動範圍內。
- **留 buffer**：每日唔好排太死；落地日 / 回程日輕量。
- **天氣 backup**：午後易雨地區，每日預留室內備案。

## Step 5 · 預算彙整
分類：交通 / 住宿 / 餐飲 / 門票 / 雜項，各有 subtotal，加 total + perPerson。
每項標可靠度；附 disclaimer（動態定價，落單前自行核實）。機票如未定可另列或標估算。

## Step 6 · 砌 tripData + 出路書
1. 按 `schema/tripData.schema.json` 砌 `trips/<目的地>-<年月>/tripData.json`。
   - 景點/酒店/餐廳盡量帶 `mapQuery`（Google Maps 搜尋字串）→ 路書先有「📍開地圖」掣。
   - `imageUrl` 只放真實 HTTPS 公開圖（官方/維基），唔肯定留空。
   - 金額用字串夾貨幣號（"HK$1,200" / "¥2,100"）。
   - 填 `essentials`（貨幣/插頭/交通卡/緊急電話/SIM/時區/語言/小費）— 旅行途中最常查。
2. 生成：
   ```bash
   python3 scripts/generate.py trips/<目的地>-<年月>/tripData.json
   ```
3. 出 `roadbook.html`（自包含、離線可用、手機優先）。

## Step 7 · 交付
- 同用戶講路書位置，叫佢用瀏覽器開（手機加到主畫面當 App 用）。
- 要改 → 改 `tripData.json` 重跑 generate.py，**唔好手改 HTML**。
- 可選：把 research 原始資料存 `trips/<目的地>-<年月>/sources/`。

---

## intake（用戶已有行程）
用戶貼 Excel / 文字 / 截圖行程：
1. 解析每日節點（D1…Dn）、景點、餐廳、酒店、已知花費。
2. OCR 截圖（如需）抽文字。
3. 補空：天氣、mapQuery、essentials、可靠度標籤。
4. 直接砌 tripData → 出路書，唔使重新由零 research（但要補導航/實用資訊等手機必需欄位）。

## 品質檢查（出路書前自查）
- [ ] 每日有 theme、活動有時間
- [ ] 景點/酒店/餐廳大部分有 mapQuery 或 address
- [ ] 預算有可靠度標籤 + disclaimer
- [ ] essentials 填齊（旅行途中速查）
- [ ] imageUrl 全部真實 HTTPS（或留空），冇作圖
- [ ] 未核實項目已標明
